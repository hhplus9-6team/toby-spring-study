# 5.1. 사용자 레벨 관리 기능 추가

## 1. [기능 추가] 파트에서의 인상깊던 부분
### 1.1. 데이터"상태"값 설정
* DB: "상태"에 대한 값을 varchar이 아닌 INT로 설정
    * → DB 용량을 많이 차지하지 않고 가볍기 때문(현재는 큰 차이 없다고 들었던 것 같다)
* App: "상태"에 대한 값을 Enum으로 설정
    * → 코드에서 가독성이 좋고, 실수로 다른 값이 들어가는 것을 방지할 수 있기 때문
    * 단, DB가 INT로 설정되어있으면 값 변환에 대해 신경써야함. 그냥 DB도 ENUM 처리하는게 좋을 듯

### 1.2. TDD 방식대로 작성했다는 것
* 테스트 코드 작성 → 실제 코드 작성 → 리팩토링
* 특히 JDBC 사용할 때 오타는 컴파일 과정에서 검증이 되지 않으므로 테스트 코드가 더 큰 위력 발휘

### 1.3. 테스트 범위 설정
* 현재 테스트는 통합테스트로 진행(이후 변경 예정이라고 언급)
* 테스트 데이터를 "경계값"으로 설정하여 테스트의 신뢰도를 높임

### 1.4. 테스트 중복 부분을 "픽처스"로 제거
```java
@Test
public void upgradeLevels() {
    // 생략
    checked(user.get(0), Level.BASIC);
    checked(user.get(1), Level.SILVER);
}
private void checked(User user, Level expectedLevel) {
    assertThat(user.getLevel()).isEqualTo(expectedLevel);
}
```
## 2. [코드 개선]파트에서 인상깊던 부분
> 1. 코드에 중복된 부분이 없는가?
> 2. 코드가 무엇을 하는 것인지 이해하기 불편하지 않는가?
> 3. 코드가 자신이 있어야 할 자리에 있는가?
> 4. 앞으로 변경이 일어난다면 어떤 것이 있을 수 있고, 그 변화에 쉽게 대응할 수 있게 작성되어있는가?

### 2.1. 분기문 개선
#### 1) 분기문을 메서드로 추출하여 [명확한 의미 부여 + 중복 제거]
```java
if (user.getLevel() == Level.BASIC && user.getLogin() >= 50) {
    user.setLevel(Level.SILVER);
    changed = true;
}
else if(user.getLevel() == Level.SILVER && user.getRecommend() >= 30) {
    user.setLevel(Level.GOLD);
    changed = true;
} else {
    changed = false;
}
```
```java
if (canUpgradeLevel(user)) {
    upgradeLevel(user);
}
```

#### 2) 분기문에서 else로 퉁치지 말고 현재 가능한 상황만을 명확히 표현
```java
private boolean canUpgradeLevel(User user) {
    if (user.getLevel() == Level.BASIC) return (user.getLogin() >= 50);
    if (user.getLevel() == Level.SILVER) return (user.getRecommend() >= 30);
    if (user.getLevel() == Level.GOLD) return false;
    throw new IllegalArgumentException("Unknown Level: " + user.getLevel());
}
```
* 현재 가능한 상황만을 명확히 표현하여, 새로운 레벨이 추가되었을 때 코드 수정이 반드시 필요함을 Exception으로 알림

### 2.2. 객체의 책임을 명확히 분리
#### 1) Enum에 변화를 Enum이 책임지도록 개선
```java
public enum Level {
    BASIC(1, null), SILVER(2, BASIC), GOLD(3, SILVER);
    private final int value;
    private final Level next;
    Level(int value, Level next) {
        this.value = value;
        this.next = next;
    }
    public int intValue() { return value; }
    public Level nextLevel() { return next; }
}
```
* Enum에 다음 레벨에 대한 정보를 추가하여, 새로운 레벨이 추가되었을 때 Enum만 수정하면 되도록 개선
* 서비스 레이어는 Enum의 nextLevel() 메서드만 호출하면 되므로, 서비스 레이어의 책임을 확실히 분리

#### 2) User 객체가 자신의 레벨을 스스로 업그레이드하도록 개선
```java
public void upgradeLevel() {
    Level nextLevel = level.nextLevel();
    if (nextLevel == null) {
        throw new IllegalStateException(this + "은 업그레이드가 불가능합니다.");
    } else {
        level = nextLevel;
    }
}
```
* User 객체가 자신의 레벨을 스스로 업그레이드하도록 개선하여, 서비스 레이어는 User 객체의 upgradeLevel() 메서드만 호출하면 되도록 개선

### 2.3. 앞으로 변경이 일어날 수 있는 부분을 추상화
#### 1) Policy 인터페이스를 만들어서, 레벨 업그레이드 정책을 추상화
```java
public interface UserLevelUpgradePolicy {
    boolean canUpgradeLevel(User user);
    void upgradeLevel(User user);
}
```
* 레벨 업그레이드 정책을 UserLevelUpgradePolicy 인터페이스로 추상화하여, 새로운 레벨 업그레이드 정책이 추가되었을 때 UserServiceImpl 클래스는 수정할 필요가 없도록 개선
